# Life Plan Lite v1.4.1

Life Plan Lite `v1.4.1` is a stable patch release on top of `v1.4.0`.

This release focuses on codebase stability and release hygiene rather than new end-user feature scope.

## Highlights

- Extracted major renderer responsibilities out of `App.tsx` into dedicated modules for list editing, widget editing, board display, layout behavior, project logic, wizard flows, modals, and navigation menus.
- Fixed the effort-duration editor so hours and minutes inputs no longer re-parse already-normalized values and multiply unexpectedly while editing.
- Synchronized the core project docs and package metadata with the real current release line.

## Included Improvements

- `App.tsx` now behaves much more like an orchestration shell instead of holding every renderer concern inline.
- List-editor support logic now lives in dedicated modules for drafts, options, structure, rows, and the editor shell.
- Widget-editor, board-display, layout-engine, project, wizard, modal, and context-menu code paths now use extracted modules as the active execution path.
- Architecture and project-context docs now describe the modular renderer state more accurately, including the remaining cleanup work in the `v1.4.x` stabilization line.
- Release metadata is now aligned across `package.json`, `package-lock.json`, roadmap/planning docs, and historical context notes.

## Fixed

- The `Effort` / duration hours field in item editing and quick-add flows no longer balloons values during input or backspace handling.
- Clearing duration inputs now returns an empty duration state instead of leaving an invalid numeric value behind.

## Release Focus

`v1.4.1` is a maintenance-focused patch:

- no new major feature surface
- safer renderer structure for continued `v1.4.x` stabilization
- cleaner foundations for the later `v2.0.0` feature line
